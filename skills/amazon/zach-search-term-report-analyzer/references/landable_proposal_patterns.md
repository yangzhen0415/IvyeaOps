# 落地化广告优化方案：结构与视觉模式

本文件总结一份高质量 **可直接执行** 的广告优化方案（来自实盘运营案例：FKPCAM FK50 trail camera 2026-05-12）所使用的结构和视觉模式。当本 skill 输出"搜索词诊断报告"，或 ops-hub 的 `html_report.py` 渲染广告诊断 HTML 时，应**对齐这些模式**以提升落地性。

> 核心差异：分析型报告只告诉你"哪个词差"，落地型方案告诉你"今天下午 3 点在哪个页面点哪个按钮，改成多少"。

---

## 整体结构（8 板块参考）

| 板块 | 作用 | 关键字段 |
|---|---|---|
| 01 现状诊断 | Campaign 级效率对比 + 核心词深度拆解 + 一句话结论 | 花费/单量/每单成本/ACOS/预算占比/单量占比/效率标签 |
| 02 核心动作 | 按优先级排列的动作清单 | 🔴P0/🟠P1/🟡P2 + 动作 / 操作路径 / 具体参数 / 预期效果 / 执行时机 |
| 03 新 Campaign 搭建 | "抄作业版"逐字段配置 | 名称/类型/日预算/竞价策略/关键词+bid/位置溢价/同步动作 |
| 04 否定词清单 | 否词 + 守护词 + 浪费金额归因 | 否定词 / 否定类型 / 原因 / 过去 N 天浪费金额($) |
| 05 加码清单 | 已验证转化词的 bid 升级 | 当前 bid / 建议 bid / 变化 % / 理由 / 预期 |
| 06 预算重分配 | Before/After 对比 + 日月预算表 | 改造前花费/占比 vs 改造后花费/占比 + 变化方向箭头 |
| 07 执行 Checklist | Day-by-day 勾选任务 + KPI 目标矩阵 | 日期 / 任务 / 操作位置 / 耗时 / ☐；KPI 当前 → 7/14/30 天目标 |
| 08 风险提示 | 绝对不做 + 警告信号对照 | 风险项 / 为什么 / 正确做法；警告信号 / 原因 / 应对 |

## 关键视觉/结构模式

### 1. Campaign 级效率表 — 而非只有关键词级
现有工具普遍在关键词粒度做分析，但**运营决策首先在 Campaign 层**。必须先看"核心词精准 65% 预算产 22% 单量 = 效率黑洞"再到词级。

效率标签：`❌ 效率黑洞` / `⚠️ 需优化` / `✓ 健康` / `✓✓ 高效`

### 2. 守护词清单 **置顶显示**
守护词是"不允许任何动作触碰的词"。不是元信息，是**一等板块**，用 🛡️ 徽章标识，高亮底色，避免后续动作误伤同义词族。
同义词族一起保护：`trail camera / trail cameras / trail cam / trail cams / trailcam` 必须作为整族保护。

### 3. 否词行带"浪费 $"归因
每条否词后面写清楚"过去 N 天因这词浪费了多少"，末尾汇总"直接省 $XX.XX"。把否词从"清理工作"升级为"ROI 明确的行动"。

### 4. Bid 建议：当前 → 建议 → 变化 %
不要只给"建议竞价"一列。要写 `$2.96 → $3.50 (+18%)`。用方向符：`⬆️+20%` / `⬇️-15%` / `➡️持平`。

### 5. 新 Campaign "抄作业版"
不是高层建议，是**精确到每个字段的配置清单**。包含：
- Campaign 名称（遵循 `SP-MatchType-Theme-Placement` 命名）
- 每日预算 / 竞价策略 / 位置溢价（搜索顶部 +X%、其余 Y%、商品页 Z%）
- 每个关键词的精确 bid
- **同步动作**：自动里要把这 N 个词加"否定精准"，防止左右手互搏

### 6. 预算重分配 Before/After
改造前每个 Campaign 花多少、占比多少 vs 改造后。必须总额持平或微增，确保用户看到"不是让我多花钱，是让钱花得更准"。

### 7. Day-by-day Checklist
Week 1 / Week 2 按日切分任务，每条带：操作位置 / 预估耗时 / ☐ 勾选框。
执行者是单人操盘手，需要知道"今天花 10 分钟做这三件事"，而不是"本周完成 X 优化"。

### 8. KPI 三时点目标矩阵
每个 KPI 列出：`当前基线 | 7 天目标 | 14 天目标 | 30 天目标`。给执行者复盘节奏感。
对 trail camera 这类冲单场景，ACOS 目标可以允许 7 天 ≤50% → 30 天 ≤40% 的渐进下降，不要一刀切。

### 9. 风险矩阵分两段
- **绝对不能做**（会破坏自然位、A9 相关性、历史数据）
- **出现这些信号立即调整**（首页位置倒退 / CTR 下降 / 日预算提前烧完）
每段都是三列：信号/风险项 · 原因 · 应对/正确做法。

## 徽章与色彩规范（可复用到 html_report.py）

| 徽章 | 语义 | 底色 |
|---|---|---|
| 🛡️ 保护 / 战略必保 | 不碰 | `#D6E8FF` (blue-boost) |
| ⬆️ 加码 | 加预算/bid | `#D8F2EC` (new) |
| ⏸️ 维持 / 观察 | 不动 | `#EFE3FF` (watch) |
| 🔪 迁移 | 从自动/广泛挪到精准 | `#FFE0C2` (cut) |
| ❌ 停 / 否定 | 立即关 | `#FBD4D4` (bad) |
| 🔴 P0 / 🟠 P1 / 🟡 P2 / 🟢 P3 | 优先级 | 红/橙/黄/绿 |

## 一句话结论的写法

每板块顶部放一句尖锐的 `verdict`，不是描述而是判断。举例对比：

❌ 弱："核心词 Campaign 花费占比较高，需关注效率"  
✅ 强："65% 预算砸在核心词 Campaign，但 CTR 0.56% = 钱烧在低质量曝光上，必须重构"

verdict 三要素：**具体数字 · 问题性质 · 必要动作**。

## 何时应用

- 用户给的目标是 `new_launch` / `relaunch` 冲量阶段：全量应用所有 8 板块
- 用户目标是 `profit` 守成：板块 03（新 Campaign）可省，但 01/02/04/05/07/08 必须有
- 用户目标是 `clearance` 清货：板块 05（加码）可省，强化 04（否词）和 06（预算收缩）

## 反模式（避免输出这些）

- ❌ 只列"高效词 Top 20 / 低效词 Top 20"然后让用户自己想怎么办
- ❌ 建议用模糊词：「考虑优化」「建议调整」「可能需要」
- ❌ 把 bid 建议写成"适当降低"而不是具体数字
- ❌ 忽略同步动作（比如自动/手动互抢导致双倍付费）
- ❌ 不区分 Campaign 级和关键词级诊断
- ❌ 把守护词埋在元信息里
- ❌ 报告结尾没有时间线，用户看完不知道下周一早上要干嘛
