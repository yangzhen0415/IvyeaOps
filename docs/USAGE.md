# IvyeaOps 使用手册（各板块说明）

本手册逐个说明 IvyeaOps 控制台的各功能板块：用途、怎么用、需要哪些配置。
安装与部署见 [`INSTALL.md`](INSTALL.md)，配置项参考见 [`CONFIG.md`](CONFIG.md)，
外部集成见 [`INTEGRATIONS.md`](INTEGRATIONS.md)。

> **标注说明**：带 🔒 的板块默认仅管理员可见，可在「用户管理」按板块授权给普通用户。

---

## 0. 先理解两个全局概念

几乎所有 AI 板块都共用同一套机制，先看懂这两点能少踩很多坑。

### 0.1 AI 降级链（所有文本 AI 的统一出口）

需要"写文本"的板块（市场调研、打法、Listing、Skill、资讯…）都走同一条**降级链**，按顺序尝试，前一个失败自动切下一个：

```
Hermes → 全局兜底大模型 → Codex → Claude
```

- **Hermes / Codex / Claude** 是本机安装的 Agent CLI（可选，首启向导可一键装 Codex/Claude）。
- **全局兜底大模型**是你在「系统配置 → 应用模型 → 全局兜底大模型」里填的任意 OpenAI 兼容模型（DeepSeek / OpenAI / OpenRouter / Kimi / 自定义…）。
  **没装任何本地 Agent 时，只要配好它，全站 AI 功能照常可用** —— 这是开箱即用的关键。
- 链的顺序可在「系统配置 → 智能体 → AI 提供商顺序」(`text_ai_providers`) 自定义。
- 健康状态见「系统配置 → 健康检查 → AI · 文本链可用 / 全局兜底 / 视觉识别」。

### 0.2 数据源（市场数据来自哪里）

首页 / 市场调研 / 打法推荐 顶部都有一个**数据源选择器**：

- **Sorftime** —— 当前唯一已接入的市场数据后端（销量、关键词、广告位、类目排行等）。需在「系统配置 → 数据源」填 `sorftime_key`。
- **SIF / 卖家精灵** —— 已预留配置位，数据客户端尚在开发中，界面标注「即将支持」，选中会显示占位说明。

## 一、选品与调研

### 首页（运营驾驶舱）
入口 `/` · 数据源驱动、**不调用大模型**

每日盯盘面板，5 个标签页：
- **大盘流量** / **关键词** / **竞品监控** / **自有 ASIN** / **类目大盘**

用法：右上角选**站点**和**数据源** → 切到对应标签 → 添加要监控的 ASIN / 关键词 / 类目 → 查看销量、排名、流量变化与告警。切换数据源会重新加载全部数据。

需要配置：`sorftime_key`。

### 市场调研 🔒不需要（对所有用户开放分析，但 AI 强制走 HTTP）
入口 `/market`

输入**关键词**或 **ASIN**，一键生成一份结构化的亚马逊市场调研报告（市场规模、竞争格局、机会与风险、SWOT、行动清单）。

用法：选「关键词/ASIN」模式 → 输入 → 选数据源(Sorftime)与站点 → **生成报告**。报告下方有：
- **深入分析**：选一个分析角度（市场诊断 / ASIN 深析 / 广告策略）+ 一个智能体，把整份报告作为文档带入「智能体会话」继续追问。
- **历史记录**：自动保存，可随时载入或导出（MD / HTML）。

AI 走标准降级链（Hermes 优先）。需要配置：`sorftime_key` + 至少一个可用文本 AI。

### 打法推荐
入口 `/playbook`

输入产品词/竞品 ASIN + **目标售价**（必填）+ 成本（选填），生成一份纯白帽站内打法手册（选品定价、关键词布局、广告结构、节奏排期、广告批量表）。

用法同市场调研，出报告后同样有**深入分析**（落地执行 / 广告细化 / 风险推演）和历史记录、`.csv` 广告批量表导出。

### 分析工具 🔒
入口 `/tools`

三组工具：
1. **ASIN 深度审计**：输入竞品 ASIN，由 Agent（hermes + MCP 抓数据）生成 COSMO/Rufus 维度的审计报告。出报告后有深入分析（Listing 优化 / 竞品策略 / 广告布局）。
2. **广告 search-term 审计**：上传 SP/SB/SD 搜索词报告 xlsx，做根因分析（竞价 / 关键词 / 结构），出报告后有深入分析。
3. **深入分析小工具**：关键词竞争、竞品流量、流量诊断、评论聚类、Listing 改写。其中「评论聚类」「Listing 改写」产出报告后也可一键带入智能体会话深挖。

> ASIN 审计 / 广告审计需要能调工具的本地 Agent（hermes）；纯兜底文本模型无法替代抓数据。

## 二、内容生产

### Listing 工作台 🔒
入口 `/listing`

从竞品 ASIN 到整套 Listing 文案 + 图片提示词的流水线：

1. **采集**：抓取竞品文案与图片。两种来源，自动择优：
   - **采集服务 `amazon-image-workflow`**（随仓库自带，`docker compose up -d` 即起，
     **免费零密钥**，curl/puppeteer 直接抓 Amazon）——安装脚本检测到 Docker 会问你是否启动；
   - 没部署它时**自动回退 Sorftime** 取产品数据。所以**没有 Docker 也能采集**，只是不走直抓。
2. **补充产品信息 / 上传自有图片**。
3. **AI 智能分析**：分析采集文案，并对**采集 + 上传的所有图片做视觉识别**（分批，覆盖每一张），提取「图片卖点清单」。
4. **生成文案**：标题 / 五点 / 搜索词 / A+，都基于上一步的卖点。
5. **生成图片提示词**：主图、副图、A+ 各槽位的生图提示词，同样融合图片卖点。

所有 AI 步骤走标准降级链；图片识别需要在「系统配置」配好**视觉模型**（Apimart Claude Vision / OpenAI / 支持视觉的全局兜底）。采集服务见 [`../amazon-image-workflow/README.md`](../amazon-image-workflow/README.md)（可选，`imgflow_url` 默认 `http://127.0.0.1:3001`）。

### AI 问答
入口 `/assistant`

直连大模型的自由对话/写作，不经过智能体、不碰文件系统，对普通用户安全。默认用「全局兜底大模型」(`assistant_*`)；留空则回退 DeepSeek→Apimart。

### AI 生图
入口 `/imagegen`

输入提示词生成图片（默认走 Apimart `gpt-image-2`，可在「系统配置 → 应用模型 → AI 生图」自定义模型/Key/Base）。生成会话本地保存，可继续追加。

## 三、运营数据

### 领星 ERP 🔒
入口 `/lingxing`

接入领星 ERP（官方 MCP / OpenAPI）查看与（受控地）操作店铺数据、广告等。配置较多，**详见专门指南** [`lingxing-erp-guide.md`](lingxing-erp-guide.md)。

### 头程比价
入口 `/freight` · **不使用 AI**

上传货代头程报价 Excel，自动解析成统一表格做横向比价（纯解析，无大模型）。

### 资讯 🔒
入口 `/news`

每日「AI 行业 + 亚马逊卖家」资讯摘要。点**立即刷新**即现场抓取 RSS 源 + 用标准 AI 链生成中文摘要（重要度、官方标记、标签），无需任何外部 cron，开箱即用。

可在「系统配置」用 `news_feeds` 自定义 RSS 源（每行 `url | 来源名 | 分类`，分类 = `ai_industry` 或 `amazon_seller`）。需要至少一个可用文本 AI。

## 四、智能体与知识

### 智能体会话 🔒
入口 `/agents`

浏览器里直接驱动本地 Agent CLI（Claude Code / Hermes / Codex…）的完整会话界面（claudecodeui 原生移植）：多会话、流式输出、工具可视化、文件管理（含**上传文件，最大 300MB**）、resume 等。市场调研/打法/分析工具的「深入分析」就是把报告作为文档带进这里继续探讨。

至少安装一个 Agent CLI 才能用（首启向导可一键装 Codex/Claude）。

### Skill 中心 🔒
入口 `/skill-hub`

「想法工坊」一句话生成 Skill：
- **创建**：输入一句话需求，走严谨的多阶段流水线（理解→规划→复核→优化→生成→自检修复）产出 SKILL.md，并可视化其 Tool Spec。
- 管理已有 Skill、从 GitHub 导入、查看执行历史、Tool 可视化执行。

Skill 生成默认走稳定的标准链（Hermes 优先 + 兜底）。

### GBrain 知识库 🔒
入口 `/brain`

本地知识库：标签页含**对话 / 上传 / 搜索 / 页面 / 模板 / 概览 / 设置**。支持把 Markdown 笔记/文档喂进去做语义检索与基于知识的问答。

需要配置 GBrain CLI（`gbrain_bin`）、知识库根目录（`brain_root`）与 Embedding 模型（系统配置 → 智能体 → GBrain Embedding）。语义检索配置要点见 `INTEGRATIONS.md`。

## 五、运维与管理

### 服务器终端 🔒
入口 `/terminal`

浏览器内的真实 PTY 终端，直接在服务器上敲命令；支持会话快照。仅管理员，谨慎授权。

### 服务器监控 🔒
入口 `/servmon`

查看服务器 CPU / 内存 / 磁盘 / 进程占用，可对高占用进程执行停止（带影响提示与二次确认）。

### 用户管理 🔒
入口 `/users`

多用户与 RBAC：新增/禁用用户、重置密码、**按板块授权**（上面带 🔒 的板块默认仅管理员，可在此逐项开放给指定用户）。

### 系统配置 🔒
入口 `/hub-settings`

全站配置中心：
- **数据源**：Sorftime / SIF / 卖家精灵 Key（带连通性测试）。
- **大模型**：Hermes 主模型 + Fallback 模型。
- **应用模型**：**全局兜底大模型（兼 AI 问答）** + AI 生图。
- **智能体**：hermes/codex/claude 路径、AI 提供商顺序、自动修复开关、GBrain Embedding。
- **飞书 / Lark 通知**、**高级 / 运维**。
- **健康检查**：一眼看清各服务与 **AI 链就绪状态**（文本链 / 全局兜底 / 视觉）。

首次使用建议先走一遍**首启向导**（含「全局兜底大模型」一步），即可让全站 AI 立即可用。
