# 广告优化方案决策原则

> 这些原则是用户长期沉淀的运营观，必须在 LLM prompt 里全部贯彻。
> 用户偶尔会让你"破例"——破例必须只针对当次任务，不写进本文档。

## 不变铁律（任何情况下都不破）

### 1. 不投 SBV 视频广告

- 用户从不投 SBV，所有 sheet 中**禁止出现 SBV 相关动作**（开启/加码/搭建）
- 即使 SIF 数据显示 SBV 表现好，也不建议——直接跳过该信号

### 2. 不走 Vine

- 用户不走 Vine 评测计划
- Sheet 02 / 08 中**禁止出现 Vine 推荐动作**

### 3. 合规底线

- 任何涉及评论操控（review manipulation）的建议必须拒绝并提供合规替代
- 例：用户提"好评截图换延保"，必须改成"主动售后跟进 + 邀请评价邮件（站内合规模板）"
- 例：用户提"刷单冲排名"，必须改成"PPC 拉量 + 低价首发"

## 优先级链（动作排序的依据）

排序规则：CTR/CVR 杠杆 > 长尾词辅推 > 出价调整 > 否词

含义：
- **CTR/CVR 杠杆**（最高优先级）：主图、Listing 文案、A+ 内容、Coupon、5 星评论补强——根因解决，影响所有 campaign
- **长尾词辅推**：自动广告挖出的金矿成交词迁精准、Phrase/Broad 拓词
- **出价调整**：bid +/- N%、位置溢价（顶部/RoS/商品页）
- **否词**：immediate / watch 否定

> 同一 ASIN 的 Sheet 02 P0 动作里，**CTR/CVR 类动作必须排在最前**，否则视为优先级倒挂。

## goal 模式定义（5 种）

LLM 收到 `goal` 字段后必须按下表选模板段：

| goal 值 | 含义 | 出价策略 | ACOS 容忍 | 模板段 |
|---|---|---|---|---|
| `profit` | 盈利为主 | 任何高花费低转化词立即砍 / 降；不为体量留亏损位置 | 严格 < 盈亏点 | 默认；保护词列表外的核心大词也按数据判断 |
| `volume` | 冲量 / 守 BSR | 适度激进，亏损位置允许保留几天看转化 | 可超盈亏点 1.2-1.5x | 关键词加码激进，否词保守 |
| `volume_with_top_of_search_push` | 冲量 + 大词必保首页顶部 | 双轨 campaign：原 campaign 兜底 + 新建 ToS-Only 专攻顶部 + 位置溢价 +150% | 接受核心大词亏损 | **5/12 FK50 模板**：sheet 02 第 1 条必为"位置溢价 +150% / RoS 0% / PP -80%"，sheet 03 必含 ToS-Only 新 campaign |
| `clearance` | 清货 | 大幅压成本，能不投就不投 | 仅做高效率词 | 否词激进，加码极少 |
| `launch` | 新品冲单 / 进 BSR | 自动广告优先放，加大长尾覆盖 | 接受 ACOS 100%+ | 自动广告全开，Phrase/Broad 拓词激进 |

## 保护词逻辑（必须严格遵守）

### 输入有保护词列表（非空）

- 保护词及其同义词在**任何 immediate / watch 否定建议中不得出现**
- 保护词的 bid 调整仍按数据建议（可降可升），只是不能否
- Sheet 04 顶部必须有 🛡️ 保护词清单红框
- Sheet 02 中涉及保护词的动作必须明确标注"保护词，不否定"

### 输入保护词为空

- **核心大词不享有 ACOS 豁免**——按数据判断该降 bid 就降、该否就否
- Sheet 04 顶部仍可放保护词区，但写"⚪ 本次未指定保护词，所有词按数据判断"
- Sheet 02 不要使用"核心大词应保护"等无条件保护表述
- 任何针对大词的保留建议必须给出具体数据证据（如"自动广告 close-match 已经在 ACOS 18% 跑出该词，可由 Auto 接管 Exact 流量"）

## 不预设产品配置

- 不要因为 ASIN 名字含"trail camera"就脑补"4G/WiFi/太阳能/IP67/电池容量"
- 仅以用户填的 `product_notes` + 上传 CSV 的实际线索为准
- 缺信息一律写"未指定"，禁止编造规格

## SIF 证据回写规则

LLM 在写 actions / boost / negatives 的 reason / expected 字段时：
- 引用 SIF 数据时必须明确标识来源，例：
  - `[SIF·需求] trail camera 处于上升期，每年增长 ~12%，建议加码`
  - `[SIF·竞争] system_state.可进入性=低，需走差异化`
  - `[SIF·关键词信号] paid_dependent，自然根基薄`
- 缺 SIF 数据时标 `[推断]`，避免以"市场看起来"开头的脑补
- CSV 数据本身的引用标 `[报告]`，例：`[报告] CTR 0.56% 低于类目基线 1.5%`

## 风险提示文案外置

Sheet 08 不进 LLM 输出 schema，由 `templates/risk_text.md` 直接渲染。
原因：所有 ASIN 通用风险（不删 campaign / 不一次降 bid >20% / 5 天稳定期 / 不否同义词）固定不变，让 LLM 重写一遍既浪费 token 又可能漂移。
